# 与信スコアリング公平性研究 面接用説明

## 30秒版

与信スコアリングモデルについて、Accuracyだけでなく、性別・年齢による公平性、SHAPによる説明可能性、別データセットへの汎化性を検証しました。Random ForestにReweighingを適用した実験では、精度を維持しながら性別バイアスを52%削減できました。この研究を通じて、高い数値をそのまま信じず、データの作り方、評価方法、利用者への影響まで確認する重要性を学びました。

## 2分版

この研究の問いは「与信モデルの精度を維持しながらバイアスを緩和できるか、さらにその効果が別データセットでも成立するか」です。

まずLogistic Regression、Random Forest、XGBoostをStratified 5-Fold Cross-Validationで比較し、AccuracyとAUCに加えてDemographic ParityとEqual Opportunityを測定しました。次にSHAPを使い、どの特徴量が予測とグループ差に影響しているかを確認しました。

改善手法としてReweighingを適用した結果、Random Forestでは性別のDemographic Parity差を52%削減し、Accuracyも2ポイント改善しました。一方で、別データセットでは同じ改善が常に再現されるわけではありませんでした。この結果から、公平性改善手法を一度適用して終わりにするのではなく、データ分布ごとに再評価し、運用中も継続監視する必要があると考えました。

技術面では、Python、pandas、scikit-learn、XGBoost、SHAPを使い、再現可能なスクリプトと実験結果をGitHubに整理しました。

## 技術選定の理由

| 選択 | 理由 |
| --- | --- |
| 5-Fold Cross-Validation | 単一分割による評価の偏りを減らすため |
| Accuracy + AUC | 正解率だけでなくランキング性能も確認するため |
| DP + EO | 予測結果の割合と、実際の正例に対する機会の公平性を分けて評価するため |
| SHAP | モデル全体と個別予測の判断根拠を説明するため |
| Reweighing | モデル本体を大きく変更せず、学習データの重みでバイアスを緩和するため |

## 難しかった点

### 精度と公平性を同じ基準で比較すること

公平性だけを改善して精度が大きく低下すると、実務で利用できません。そこで、改善前後でAccuracy、AUC、DP、EOを同時に記録し、トレードオフを表で確認しました。

### 高い評価値が本当に信頼できるか判断すること

前処理の順序やデータ分割によってデータリーケージが起こる可能性があります。PipelineとFold単位の学習を意識し、評価データの情報が学習に入らないようにしました。

### 改善結果の汎化性

German Creditで有効な方法がSouth German Creditでも同じように機能するか確認しました。結果が完全には一致しなかったため、データ分布が変わると公平性も再評価が必要だと分かりました。

## 研究の限界

- 公平性の定義は一つではなく、DPとEOだけですべてを判断できません。
- データセットは実際の金融機関の最新運用データではありません。
- 性別・年齢以外の交差属性については、より詳細な検証が必要です。
- 公平性閾値は業務・法務・社会的合意と合わせて決める必要があります。
- モデルの導入には、人によるレビューと継続的なモニタリングが必要です。

## 想定質問

### なぜAccuracyが下がっても良い場合がありますか

過学習やデータリーケージによる高いAccuracyは本番で再現できません。少し低くても、未知データで安定し、判断根拠を説明でき、公平性を継続監視できるモデルの方が実務価値は高い場合があります。

### 公平性と精度が衝突したらどうしますか

一つの指標だけで自動決定せず、複数の閾値候補を比較し、業務担当・法務・リスク管理と許容範囲を決めます。影響の大きい判断では人によるレビューも残します。

### 実運用へ進める場合、何を追加しますか

データドリフト監視、グループ別性能監視、説明ログ、モデルバージョン管理、異議申立てフロー、人によるレビューを追加します。
